📚 EVV Report Generator Pro v4.2 — Complete Guide

🎯 Purpose

This application merges EVV billing data with slot (scheduling) data and client information to produce a validated billing report. It detects discrepancies, identifies unbilled services, verifies claim generation, handles multi-provider scenarios, and auto-corrects known data quality issues.

📁 Required Files

  1. Billing Data File — From Therap: Billing > Billing Data > Search. Filename should contain Billing_Data.
  2. Slot Report File — From Therap: Agency Reports > Report Library > Detailed Slot Report. Filename should contain Detailed_Slot. The app auto-deduplicates by Form ID.
  3. Client By Status File — Contains MAIDs and Funding Sources. From Focus or your shared drive.
  4. Transaction Search File (optional, for Claims Check) — The claims export. Column B must be named Individual. Filename should contain Transaction_Search.

🚀 How to Use

  1. Load Files: Use Browse buttons or drag all files into the drop zone. Files are automatically identified by filename pattern.
  2. Select Output Mode: Full includes all columns; Basic uses your customizable column list.
  3. Merge Data: Click Merge — the app processes all files, runs fuzzy matching, deduplicates slot data by Form ID, applies Slot Start Date fallback, and auto-corrects funding sources.
  4. Review Alerts: Post-merge, the Merge tab shows alert banners if unbilled services or missing claims are detected.
  5. Unbilled Services Tab: Shows slot records with no matching billing entry, including unclosed visits (missing check-out) and estimated revenue impact.
  6. Claims Check Tab: Shows individuals in the Slot Report not found in Transaction Search — a strong indicator of claim generation failure.
  7. Export Reports: Save the main Excel report from the Merge tab, or individual reports from each analysis tab.

✨ New in v4.2

  • Transaction Search Integration — 4th file input enables Claims Check verification
  • Claims Check Tab — Detects individuals present in Slot Report but absent from claims, flagging likely billing failures
  • Unbilled Services Tab — Full unbilled analysis with unclosed visit detection and revenue impact estimation
  • Form ID Deduplication — Slot data is automatically deduplicated using Form ID before processing
  • Slot Start Date Fallback — When Billed Service 1 Date is empty, Slot Start Date is used automatically
  • Post-Merge Alert Banners — Prominent alerts on Merge tab if issues are detected after processing
  • Unclosed Visit Detection — Slots missing a check-out date are flagged with ⚠ UNCLOSED status

🔧 Automatic Data Corrections

  • Medicare/Medicaid cleanup — Multi-source funding strings are auto-corrected
  • Fuzzy name matching — 85% similarity threshold handles minor name variations
  • Multi-provider row splitting — Rows with multiple providers are split with units allocated by actual time worked
  • Date normalization — Handles Excel serial dates, multiple string formats, and missing values

🔒 Data Privacy

All processing runs entirely in your browser. No data is ever transmitted to any server. This application is fully HIPAA-compliant for local use.
